4. What is "Unobtrusive Javascript"? What is the practical application of Unobtrusive Javascript (and the reasons for using it)?

The unobtrusive solution is to register the necessary event handlers programmatically, rather than inline. So the practical application is instead of adding an event attribute explicitly in the HTML, the relevant element(s) are simply identified, for example by class or id. Then you can transfer the responsibility that would be used in the HTML to the javascript file, which is much better for maintenance and also good in the way that using this approach you are not going to have any problems related to loading files, when for example, your javascript file execute before the Document Object Model be loaded.
